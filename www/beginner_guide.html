Beginner guide

The README in repository holds out everything you need to get yourself a working development environment. But what next? What do you do with this thing? Here's a small guide to help you out.

Tundra mode

Many other languages have tens to hundreds of contributors, or they pretend they have. Lever has only one author who is very aware of what he is capable of alone.

There are very few resources, utilize them to maximum.

Semantics documentation

You should get understanding of how to piece up programs to use Lever. I have provided lot of sample programs but they aren't much of help if you don't know how the language is constructed.

semantics_documentation.txt document the semantics of Lever source files. This file describes every semantic construct supported by Lever. In combination with the grammar it defines the Lever programming language.

Grammar files

lever-0.8.0.grammar holds a context-free grammar for Lever. This grammar describes how syntax maps to semantics.

Internal documentation

You should glance the internal_documentation.txt through. It tells how everything works within the runtime. There's also quick-reference for all built-in functions.

I wish you well.
